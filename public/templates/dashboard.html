Recent Updates to Roadmapper

The following is the changelog for the roadmapper tool. Want to see some more improvements? Open an issue on Github!

December 27, 2013

  • Pagination for features and problems
  • Removed modal dialog boxes in favor of separate pages
  • Bonus: you can now open problems and features in new windows (Cmd-Click)
  • Forms can be quickly submitted using Cmd-Enter
  • The features table will hide the state, quarter, and team columns if you are searching by those criteria
  • The UI now works much better on an iPhone 5 and iPad
  • Option to quickly "create another" problem, useful for rapid entry from customer support
  • When browsing features and problems you can quickly jump to the previous/next
  • Improved performance in many areas

September 11, 2013

  • First pass at a tag management UI
  • Internal improvement: removed password field from users table since we're 100% on OAuth now

September 8, 2013

  • Teams can now have customizable utilization rates rather than the default of 40%
  • Cache busting logic should help with Firefox and Chrome bugs reported
  • Internal errors will at least report a generic error now, rather than a blank red box
  • Global stats showing off activity this week and last week
  • Much improved table design in problems and features -- use your whole screen real-estate!
  • Resolved some (but not all) glitches with autocomplete race condition issues
